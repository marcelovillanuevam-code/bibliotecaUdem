📁 Artefactos de Trabajo
Sistema de Biblioteca Inteligente UDEM con Chatbot Integrado

Este repositorio contiene la documentación y evidencias generadas durante el desarrollo del proyecto correspondiente a la materia de Metodologías de Desarrollo de Software.

El objetivo de esta carpeta es centralizar todos los entregables del proyecto, permitiendo su consulta, revisión y validación por parte de los evaluadores.

📌 Contenido del Repositorio

A continuación se listan los artefactos incluidos:

📄 Matriz de Rastreabilidad del Proyecto
Relación entre requerimientos, casos de uso y funcionalidades desarrolladas.
⚠️ Análisis de Riesgos
Identificación, evaluación y mitigación de riesgos del proyecto.
📋 Sprint Backlogs
Planeación detallada de actividades realizadas en cada sprint.
👤 Casos de Uso
Descripción completa de las funcionalidades del sistema desde la perspectiva del usuario.
🎨 Prototipo Completo
Diseño visual e interacción del sistema (interfaces y navegación).
🧩 Diagrama de Clases
Representación estructural del sistema y sus relaciones.
🗄️ Base de Datos Customizada
Diseño y estructura de la base de datos utilizada en el sistema.
🔄 Diagramas de Flujo de Datos (DFD)
Representación del flujo de información en el sistema (Nivel 0 y Nivel 1).
💻 Aplicación Construida
Implementación funcional del sistema desarrollada durante el proyecto.
✍️ Firmas de Aceptación de la Propuesta de Negocio
Documento firmado que valida la propuesta inicial del proyecto.
📑 Carta de Aceptación de la Primera Entrega
Documento firmado que valida la entrega inicial del proyecto.
📝 Minutas de Junta
Registro de reuniones realizadas durante el desarrollo del proyecto.
🚀 Propuestas de Mejora
Sugerencias de mejora generadas después de cada entrega.
🔓 Acceso

Este repositorio cuenta con los permisos necesarios para su correcta visualización. Todos los archivos pueden ser consultados libremente para fines académicos y de evaluación.

📚 Nota

Todos los documentos fueron desarrollados como parte del proceso iterativo del proyecto, siguiendo la metodología SCRUM y considerando las necesidades del Sistema de Biblioteca Inteligente UDEM.
